# 🤖 Agentic RAG Backend

A highly capable, production-ready AI Agentic Retrieval-Augmented Generation (RAG) backend built with **FastAPI**. 

This system processes PDF documents, chunk-embeds them using FAISS and SentenceTransformers locally, and intelligently extracts precise data using **Gemini 2.5 Flash** to natively return pure, structured JSON arrays back to your frontend applications.

## ✨ Key Features

- **Intelligent Multi-Query Routing**: The agent understands complex, compound questions (e.g., *"What is the name? What is the phone number?"*) and autonomously splits them into individual sub-queries, drastically increasing context retrieval performance against disparate document chunks!
- **Local FAISS Vector Database**: Fast, in-memory dense vector similarity search running entirely locally using lightweight `all-MiniLM-L6-v2` embeddings.
- **Structured JSON Extraction**: Enforces strict instructions for the LLM to extract and synthesize required data directly into cleanly mapped JSON formats rather than generic conversational blobs.
- **Modern SDK Integration**: Fully upgraded to leverage the official `google.genai` SDK and the `gemini-2.5-flash` model.

## 🚀 Getting Started

### Prerequisites

1. Ensure you have Python 3.10+ installed.
2. Obtain a Gemini API Key from [Google AI Studio](https://aistudio.google.com/app/apikey).

### Installation

1. Create a virtual environment and load the required dependencies:
```bash
pip install -r requirements.txt
```

2. Create a `.env` file in the root directory mirroring `.env.example`:
```env
GEMINI_API_KEY="AIzaSyYourKeyHere..."
```

### Running the Server

Start up the Uvicorn server:
```bash
uvicorn app.main:app --reload
```

You can then test the system immediately by navigating to the interactive **Swagger UI** generated by FastAPI at `http://127.0.0.1:8000/docs`.

## 📡 API Endpoints

### `POST /upload`
Accepts a binary PDF document via `multipart/form-data`, chunks the text using a `RecursiveCharacterTextSplitter`, embeds it, and saves it into the local FAISS index.

### `POST /query`
Accepts a query string, triggers the multi-query extraction pipeline to hit the vector store independently for each subset of the question, drops the aggregated chunks into the Gemini LLM window, and responds with a cleanly structured JSON extraction dictionary!

**Example Request:**
`POST /query?query=What is the candidate name? What is the email address? What is the phone number? What is the highest qualification?`

**Example Response:**
```json
{
  "candidate_name": "SHASHIDHAR REDDY D",
  "email_address": "shashidharreddy893@gmail.com",
  "phone_number": "+91 6301670751",
  "highest_qualification": "B.Tech - Electronics and Communication Engineering"
}
```
